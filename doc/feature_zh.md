# UseUp - 功能文档
**版本：1.0.2**

UseUp 是一款基于 Flutter 开发的个人库存管理和保质期追踪应用。它旨在帮助用户减少食物浪费，高效管理家庭库存物品。

## 核心功能

### 1. 智能仪表盘 (Smart Dashboard)
应用的控制中心，提供库存概况的一目了然的视图。
*   **可视化过期追踪：** 基于过期状态对物品进行颜色编码（例如：绿色表示安全，橙色表示即将过期，红色表示已过期）。
*   **智能筛选：** 
    *   **搜索：** 支持对物品名称进行即时文本搜索。
    *   **多维过滤：** 可按具体的 **分类 (Categories)**（如：乳制品、肉类）或 **存放位置 (Locations)**（如：冰箱、储藏室）筛选库存。
    *   **快捷标签：** 快速切换“全部物品”、“即将过期”和“已过期”视图。
*   **快捷操作：** 
    *   **滑动手势：** 向左滑动 **删除**，向右滑动 **标记为已使用 (Consume)**。
    *   **临期置顶：** 顶部横向滚动区域突出显示即将过期的物品，提醒用户优先处理。

### 2. 高级物品管理 (Advanced Item Management)
强大的物品录入与编辑系统，支持多种追踪方式。
*   **双重录入模式：**
    *   **过期日期模式：** 直接选择确切的过期日期。
    *   **生产日期模式：** 输入“生产日期 + 保质期”（例如：“18个月”），系统自动计算过期日期。
*   **丰富细节：**
    *   **图片支持：** 支持通过相机拍摄或相册选择物品图片。
    *   **组织管理：** 支持分配自定义的分类和存放位置。
    *   **多计量单位：** 支持多种单位（个/pcs, 千克/kg, 克/g, 升/L, 毫升/ml, 包/pack, 盒/box）。
    *   **价格记录：** 记录物品购买成本。
*   **自定义提醒：** 用户可以为同一物品设置多个提醒时间点（例如：在过期前 1 周和前 1 天分别发送通知）。

### 3. 历史记录与补货 (History & Restocking)
追踪使用习惯并简化再次购买流程。
*   **消耗日志：** 被标记为“已使用”的物品会移入历史记录页面，保持主库存列表整洁。
*   **一键补货：** 用户可以从历史列表中快速重新添加物品。系统会自动预填该产品的过往详情（如名称、分类、保质期偏好），大大节省录入时间。

### 4. 本地化 (Localization)
*   **多语言支持：** 应用完全支持 **中文** 和 **英文**，界面文本、计量单位及货币符号会自动根据系统语言适配。

## 技术栈

*   **框架：** Flutter (Mobile - iOS/Android)
*   **架构：** Riverpod (状态管理) + GoRouter (路由导航)
*   **数据库：** Isar (高性能本地 NoSQL 数据库)
*   **通知：** Flutter Local Notifications (本地定时提醒)
*   **UI/UX：** Material 3 设计风格，采用清新的绿色/中性色调。

## 数据模型

### 物品实体 (Item Entity)
*   `name`: 物品名称
*   `quantity` / `unit`: 数量与单位追踪
*   `expiryDate`: 用于通知提醒的关键日期
*   `productionDate` / `shelfLife`: 辅助追踪的元数据（生产日期/保质期）
*   `category` / `location`: 用于组织管理的关联关系
*   `imagePath`: 本地图片文件路径
*   `notifyDaysList`: 提醒设置列表（存储距离过期还有多少天时触发提醒的整数列表）。
